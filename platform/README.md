# Root `platform/` directory

Two distinct things live here:

- **`crossplane/`** — render output. CI-generated by `make export-cluster CLUSTER=demo` from `framework/`. Contains the XRDs and Compositions every tier 3+ folder references. **Do not hand-edit.**
- **`external-secrets/`, `kubescape/`, `kyverno/`, `headlamp/`** — hub-cluster bootstrap installs. These are NOT consumed by any tier folder; they exist for the hub-cluster operator that runs `make apply CLUSTER=demo` against a fresh cluster and needs ESO/Kyverno/Kubescape installed before Crossplane Compositions can reconcile. The tier folders include their own `platform/{traefik,cert-manager}/` and (at tier 1+) `platform/kube-prometheus-stack/`, `platform/loki/`, etc.

## Why these aren't in `infrastructure/`

`infrastructure/` is for **version-pinned manifests of the substrate Crossplane needs to function** (provider-helm, provider-kubernetes, the Crossplane installer itself). The four hub-bootstrap installs here are a layer above — they're operators that the framework's Compositions assume are present, but they're not Crossplane substrate.

The cleanest answer at the SE conversation is: at tier 3+ the platform team installs ESO, Kyverno, etc. via Argo CD Applications inside the relevant tier folder (see `<tier>/platform/external-secrets/` if/when wired). This root directory is the bootstrap shortcut for the hub cluster only.
